# Свои — объявления Луганска

Локальная доска объявлений в стиле Avito/Юла, заточенная под Луганск. React + TypeScript + Vite + Capacitor 6 + Zustand на фронтенде, Supabase (Postgres + Auth + Storage + Realtime) в качестве бэкенда.

## Что уже сделано

- Полная структура экранов: лента, поиск, карточка объявления, создание объявления с загрузкой фото, чаты с realtime-сообщениями, профиль, вход по SMS.
- Дизайн-система в `src/styles/tokens.css` — тёплая палитра (не бирюзово-корпоративная, не Avito-синяя), фирменный элемент — цена в виде «рыночного ценника».
- Полная SQL-схема Supabase в `supabase/schema.sql`: таблицы, индексы, RLS-политики, сиды категорий, storage-бакеты.
- Иконка приложения обработана и встроена во все плотности (`mipmap-mdpi` … `mipmap-xxxhdpi`), включая adaptive icon (foreground/background) и legacy/round варианты — лежит в `android/app/src/main/res`.
- Android-платформа развёрнута через Capacitor (`npx cap add android`), проект собирается (`npx tsc --noEmit` и `npx vite build` проходят без ошибок).

## Что нового

- **Телефон в объявлении**: при создании объявления теперь обязательно указывается номер — он подтягивается из профиля, если уже сохранён, либо вводится заново. На странице объявления рядом с «Написать продавцу» появилась кнопка **«Позвонить»** (открывает звонок напрямую, как на Авито).
- **Если ваш Supabase-проект уже создан и `schema.sql` уже накатан** — не запускайте его заново (упадёт на «уже существует»). Вместо этого выполните в SQL Editor только `supabase/migration_001_contact_phone.sql` — это одна строка, добавляет недостающую колонку.
- **Красивый фон** добавлен на все экраны — мягкий градиент в тон палитре (тёплый + бирюзовый + marigold), не отвлекает от контента.

## Как запустить

1. Создайте проект на [supabase.com](https://supabase.com), выполните `supabase/schema.sql` целиком в SQL Editor.
2. **Вход по email, с подтверждением** (это настройка по умолчанию — трогать ничего не нужно): Authentication → Providers → Email → **Confirm email** остаётся включённым. Так регистрация мусорными/чужими адресами отсекается сама собой.
3. Скопируйте `.env.example` → `.env`, впишите `VITE_SUPABASE_URL` и `VITE_SUPABASE_ANON_KEY` из Project Settings → API.
4. `npm install`
5. `npm run dev` — веб-версия на `localhost:5173` для быстрой разработки.
6. `npm run build && npx cap sync android` — собрать веб-бандл и обновить нативный проект.
7. `npx cap open android` (нужен Android Studio) — открыть проект для сборки APK/AAB и публикации в RuStore.

**Как выглядит вход для пользователя:** заполняет имя/email/пароль → приложение показывает «Проверьте почту и перейдите по ссылке» → человек подтверждает почту в браузере → возвращается в приложение и просто жмёт «Войти» тем же email/паролем. Код уже полностью поддерживает этот сценарий — ничего дополнительно писать не нужно.

**Про вход:** сейчас в приложении только email + пароль — без SMS (это всегда платно у любого провайдера — Twilio, MessageBird и т.д., бесплатного пути отправлять настоящие SMS не существует) и без VK ID (у него нестандартный OAuth-флоу с обязательным `device_id`, который плохо ложится на генерик Custom OAuth Provider в Supabase — решили не усложнять). Email через встроенный сервис Supabase бесплатен в пределах бесплатного тарифа.


## Что осталось сделать перед публикацией

- **Модерация**: сейчас все новые объявления получают статус `pending_review` и не попадают в ленту, пока вы вручную не переключите `status` на `active` в Supabase Table Editor. На старте это нормально (ручная модерация), позже можно сделать админ-экран.
- **Реклама**: интеграция Yandex RSY по тому же пайплайну, что в ПДД Тренажёре — баннер в ленте между карточками, интерстишл при переходе в объявление.
- **Холодный старт**: наполните ленту вручную первыми объявлениями (см. обсуждение про перенос из локальных ВК/Telegram-групп), иначе первым пользователям будет нечего смотреть.
- **RuStore listing**: иконка для карточки магазина лежит в `assets/store/icon_512.png` и `icon_master_1024.png`.

## Структура проекта

```
src/
  components/    — BottomNav, TopBar, ListingCard, CategoryChip, FAB
  screens/       — 8 экранов (лента, поиск, объявление, создание, чаты×2, профиль, вход)
  store/         — 3 Zustand-стора (auth, listings, chat)
  lib/supabase.ts
  types/index.ts — типы, зеркалящие схему БД
supabase/schema.sql — вся база одним файлом
android/         — нативный проект Capacitor с встроенной иконкой
assets/store/    — иконка для карточки RuStore
```
